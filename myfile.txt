- 원숭이도 이해할 수 있는 Git 명령어 - 과연?
add: 변경 사항 만들어서 인덱스에 등록한다
commit: 인덱스 상태를 기록
pull: 원격 저장소 내용 가져오기
tag: 커밋을 참조하기 쉽게 이름 지정
merge: 변경 이력을 합침
--squash: 커밋 내용을 하나로 병합하여 merge실행

git status : 상태확인은 늘 필요합니다.
git checkout -t <branch_name>: 원격에 존재하는 브랜치를 로컬로 가져올 수 있다.
git branch: 새로운 브랜치 생성
